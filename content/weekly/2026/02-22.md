    - コーディングエディタ用のインテグレーションとかもあって開発にも使えるってことで試してみた
  - 音声入力、やっぱ文字入力よりも圧倒的に早いので本当に便利
    - 最近は開発も自然言語になっているのでそこととても相性が良い。というかだからこそ音声入力ソフトがたくさん出ているわけで
  - 音声入力起動中に裏で画面収録をしていて、今表示している画面を見てコンテキストを理解して文字起こししてくれているらしく、それはすごいなと
  - プログラミングやドキュメントやメッセージを作るっていうような仕事で便利な使い道以外にも、寝る前に音声日記みたいなノリでマイクに向かって話して雑にその日の感情を残しておく、とかをやってみている
    - しばらくやってみてなんか意味があるのか様子を見てみようと思う
  - マイクを常に口元に向けてないといけない問題というのがあるので、これの解決方法を見つけたい

## 振り返り

- mac と Windows で開発環境の統一を図っている
  - 先週新しい開発環境を mac で構築したが、これと全く同じように Windows で開発できるよう dotfiles を修正したりしている
  - Windows とは言っても WSL の中で Ubuntu を使うって感じなので実際はそこまで変化はない
  - mac と Windows どっちも使うってのは僕の中でこだわりとしてあって、仕事は強制で mac を使わざるを得ないので、プライベートではあえて Windows にこだわり続けたい
- キャパシティの限界を感じている
  - 仕事のプロダクト開発の忙しさとしてはいつも通りって感じなんだけど、今年の TSKaigi という技術カンファレンスに向けた自身のプロポーザルの準備だったり社内のメンバーのプロポーザルの支援だったり、スポンサー運営のとりまとめだったりをやっていて、キャパシティを超えている感じがする
  - 常に複数のやらなきゃいけないことがあると、マルチタスクが苦手だとかスイッチングコストがかかるとかそれ以前に、頭がパンク状態で何もせずぼぉ〜っと過ごす時間が増えてしまう感じがする
  - 本当は自分の作っているプロダクトのことだけをずっと考えていたいけど、責任ある立場になるためにはそういうわけにも行かなくて、そして結局自分の仕事のできなさを感じる。キャパシティが小さすぎる
